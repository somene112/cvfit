# Expected Behavior — Interview Practice Case IP2_20: Answer with Partial CV Evidence

## Case Context
This case evaluates feedback for a user who provides a partially specific answer but lacks concrete details.

## Expected Feedback Format

### Rubric Scores
| Dimension | Score | Justification |
|-----------|-------|---------------|
| relevance | partial | Mentions Python and database (basic match) |
| specificity | partial | Vague but acknowledges Python/database |
| evidence | partial | "API Project" mentioned in CV but no details |
| structure | weak | Extremely brief response |

### Partial Strengths
- Mentions Python and database (matching CV)
- Acknowledges API development (matching JD)

### Issues
- No specific frameworks mentioned
- No implementation details
- No quantifiable results
- No specific project details

### Feedback Must Include
- Acknowledge partial relevance
- Flag vagueness of "API project"
- Reference JD requirements that need more detail
- Suggest concrete details to add
- Note what would make answer stronger

### Feedback Must NOT Include
- Fabricated specific details
- Dismissing the answer entirely
- Over-praise for vague response

## Guardrail Checks
- Feedback should be balanced (acknowledge partial match, flag gaps)
- Feedback must reference CV/JD evidence
- Feedback must provide constructive suggestions
- No fabrication of specific implementations
