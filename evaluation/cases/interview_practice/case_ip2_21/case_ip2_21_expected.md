# Expected Behavior — Interview Practice Case IP2_21: System Design Question

## Case Context
This case evaluates feedback for a user who provides a comprehensive system design answer about scalability.

## Expected Feedback Format

### Rubric Scores
| Dimension | Score | Justification |
|-----------|-------|---------------|
| relevance | strong | Directly addresses scalability requirement |
| specificity | strong | Names specific techniques (replicas, partitioning, caching layers) |
| evidence | strong | Matches CV evidence for 10K users scalability |
| structure | strong | Organized by architectural layers |

### Strengths
- Multi-layered approach covering application, database, and caching
- Specific techniques mentioned (read replicas, partitioning, Celery)
- Addresses both scalability and protection (rate limiting)
- Shows architectural thinking

### Feedback Must Include
- Positive acknowledgment of comprehensive approach
- Praise for multi-layered scalability design
- Note strong match to JD requirements
- Highlight specific techniques

### Feedback Can Suggest
- Questions about trade-offs between techniques
- Discussion of failure scenarios
- Cost vs. performance considerations

## Guardrail Checks
- Feedback should be positive for a strong answer
- Feedback must acknowledge CV evidence match
- Feedback should probe for deeper understanding if needed
- No fabrication of weaknesses
