# Expected Behavior — Interview Practice Case IP2_17: Gap Probe — Missing Skill (FastAPI)

## Case Context
This case evaluates feedback for a user who honestly acknowledges lacking FastAPI experience. The CV shows Flask experience but no FastAPI.

## Expected Feedback Format

### Rubric Scores
| Dimension | Score | Justification |
|-----------|-------|---------------|
| relevance | moderate | Acknowledges gap honestly |
| specificity | moderate | Shows basic FastAPI awareness |
| evidence | partial | Flask experience transferable but FastAPI gap acknowledged |
| structure | strong | Clear and honest acknowledgment |

### Positive Aspects
- Honest acknowledgment of FastAPI gap
- Shows proactive learning initiative
- Demonstrates understanding of the gap
- Expresses willingness to learn

### Feedback Must Include
- Acknowledge honesty positively
- Reference Flask experience as transferable foundation
- Provide learning roadmap for FastAPI
- Note that Flask async is a good starting point
- Suggest specific FastAPI learning resources
- Encourage continued learning

### Feedback Must NOT Include
- Harsh criticism for missing skill
- Fabricated FastAPI experience
- Dismissing Flask experience

## Guardrail Checks
- Feedback must acknowledge honesty
- Feedback must validate transferable skills
- Feedback must provide constructive guidance
- No fabrication of FastAPI experience
- Feedback should be encouraging
