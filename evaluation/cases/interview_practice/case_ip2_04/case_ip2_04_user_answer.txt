For the query optimization case, I used PostgreSQL's EXPLAIN ANALYZE to identify bottlenecks. The main issues were missing indexes on foreign key columns and inefficient JOIN conditions. I added B-tree indexes on frequently queried columns and rewrote the JOIN using subqueries with appropriate filtering. I also implemented partial indexes for common query patterns. The result was a query that went from 45 seconds to under 500ms. I then created a monitoring system using pg_stat_statements to track slow queries weekly and automatically alert when queries exceeded 1 second.
