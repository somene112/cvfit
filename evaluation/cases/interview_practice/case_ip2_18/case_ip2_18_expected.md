# Expected Behavior — Interview Practice Case IP2_18: Gap Probe — Honest About Gap

## Case Context
This case evaluates feedback for a user who honestly acknowledges lacking AWS production experience but shows initiative in learning.

## Expected Feedback Format

### Rubric Scores
| Dimension | Score | Justification |
|-----------|-------|---------------|
| relevance | moderate | Acknowledges AWS gap honestly |
| specificity | moderate | Shows awareness of specific AWS services |
| evidence | partial | Training knowledge but no production AWS experience |
| structure | strong | Clear and self-aware response |

### Positive Aspects
- Honest about learning vs. production experience
- Shows awareness of the gap
- Demonstrates initiative in training
- Realistic about skill level

### Feedback Must Include
- Acknowledge honesty and self-awareness positively
- Reference CV evidence (Docker basics as related skill)
- Provide learning path for gaining AWS production experience
- Suggest projects to build AWS portfolio
- Note transferable skills from current role
- Encourage continued learning

### Feedback Must NOT Include
- Fabricated AWS production experience
- Dismissing training as irrelevant
- Harsh criticism for the gap

## Guardrail Checks
- Feedback must validate honest acknowledgment
- Feedback must provide constructive guidance
- Feedback must not fabricate experience
- Feedback should be encouraging and realistic
