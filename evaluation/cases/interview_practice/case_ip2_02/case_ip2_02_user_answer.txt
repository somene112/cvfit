Regarding the PostgreSQL sharding implementation, I designed a horizontal sharding strategy based on user_id hash. We split the database into 8 shards, each running on separate RDS instances. I implemented a custom shard router in Python that determines which shard to query based on the hash value. This reduced our database load by 75% and improved query latency from 500ms to under 50ms for user-specific queries. We also implemented cross-shard queries for analytics using a separate read replica, which didn't affect the main write operations. The key challenges were ensuring data consistency during the migration and handling the case where a user's data needed to be in a specific shard for compliance reasons.
