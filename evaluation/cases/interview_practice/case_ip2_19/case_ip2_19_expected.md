# Expected Behavior — Interview Practice Case IP2_19: Strong Answer with CV Evidence

## Case Context
This case evaluates feedback for a user who provides a detailed answer that closely matches their CV evidence.

## Expected Feedback Format

### Rubric Scores
| Dimension | Score | Justification |
|-----------|-------|---------------|
| relevance | strong | Directly addresses all JD requirements |
| specificity | strong | Names specific implementations and metrics |
| evidence | strong | Closely matches CV evidence |
| structure | strong | Well-organized with clear technical details |

### Strengths
- Specific technical implementations (B-tree indexes, partial indexes)
- Quantifiable metrics matching CV (40% reduction, 72ms response time)
- Demonstrates understanding of FastAPI async patterns
- Shows testing best practices (pytest, 85% coverage)

### Feedback Must Include
- Positive acknowledgment of strong alignment with CV
- Praise for specific technical details
- Note the strong match to JD requirements
- Highlight quantitative results

### Feedback Can Suggest
- Potential areas for deeper discussion
- Questions about trade-offs or alternatives

## Guardrail Checks
- Feedback should be positive and encouraging
- Feedback must acknowledge CV evidence match
- Feedback must not fabricate weaknesses
- No excessive criticism for a strong answer
